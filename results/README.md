# Result files

The committed result tree separates observations used for inference from generated summaries.

## Canonical tables

For each dataset and decoder prefix:

- `*_results.csv`: fold/repeat measurements produced by the benchmark runner.
- `*_subject_summary.csv`: one row per participant and condition; this is the inferential input.
- `*_population_summary.csv`: descriptive condition means and bootstrap intervals.

The PhysioNet decoder comparison is stored in:

- `PhysionetMI_csp_lda_vs_riemann_lr_paired_comparison.csv`
- `PhysionetMI_csp_lda_vs_riemann_lr_paired_subject_differences.csv`

Other statistical tables are regenerated by `make statistical-reports` and written under `artifacts/generated_reports/` or, for machine-readable checks, under `artifacts/validation/` and `artifacts/manifests/`.

## Key columns

| Column | Meaning |
|---|---|
| `subject` | Dataset participant identifier |
| `pipeline` | `csp_lda` or `riemann_lr` |
| `stressor` | Clean, random dropout, regional dropout, reduced montage, or cross-session |
| `dropout_fraction` | Fraction of channels set to zero; zero for clean and reduced montages |
| `region` | Named channel region when regional dropout is used |
| `roc_auc` | Receiver operating characteristic area under the curve |
| `balanced_accuracy` | Mean class recall |
| `brier_score` | Mean squared probability error |
| `ece` | Ten-bin expected calibration error |

Raw EEG is not included. See `DATA_PROVENANCE.md` for access terms.
